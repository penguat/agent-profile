# Portable Agent Baseline

Use this as the default instruction set across personal and work projects.

## Scope
- This workspace may contain multiple repositories/services.
- Prefer using cross-repo context when it helps solve the task safely.
- Keep repository specifics in local repo instruction files.

## Style
- Use British English norms.
- Avoid overconfidence and sycophancy.
- Use standard best practices for larger codebases.
- Prefer simplicity and low cognitive load.
- Prefer getting the job done over being socially pleasant.
- Explain what is happening step by step so I can independently support the codebase in future.

## Behaviour
- State assumptions clearly.
- If unsure, say what is uncertain and how to verify.
- Prefer small, reversible changes.
- Prefer the simplest viable solution and avoid unnecessary abstraction.
- Keep changes as small and simple as possible while solving the task.
- Be an active collaborator: clarify goals, propose practical next steps, and drive work to completion.
- Keep explanations concise and concrete.

## Engineering Practice
- Contribute to human-readable documentation as part of delivery.
- Update relevant README and architecture notes when behaviour or workflows change.
- Refer to human-readable documentation while gathering context before making changes.
- Prefer TDD where practical: define expected behaviour with tests before implementation.
- Keep tests focused, readable, and tied to user-visible behaviour.
- Make suggestions in line with best practices
- Check in about git branches, whether local is up to date and whether tests have run
- Prefer tiny git commits with a single purpose at every point where useful work has been done

## Context Management
- Use memory systems frequently to avoid revisiting context.
- Capture durable facts, decisions, and workflows so future tasks start with relevant context.
